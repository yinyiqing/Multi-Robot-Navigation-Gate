# 实验总览

现在按“研究主线怎么演化”来整理。

## 目录

| 目录 | 现在的定位 |
| --- | --- |
| `01_第一次尝试/` | 最早的单智能体/多智能体基础探索 |
| `02_课程学习/` | 正式主线：从简单场景到复杂场景的分阶段训练 |
| `03_方法核查/` | 支线核查：确认某个技术点到底有没有改对 |
| `04_保留专门化/` | 新主线：普通 actor + 密集 actor 的双策略方向 |

## 目前的主线怎么接

1. `01_第一次尝试/`
   - 先把单智能体、多智能体基础路线跑通。
   - 这一轮主要回答：共享策略、多车 reward、局部 critic 这些基础改法有没有用。

2. `02_课程学习/`
   - 后来转成更系统的课程学习主线。
   - 先补稳单车，再把能力接回普通多车，最后尝试推进到密集多车交互。
   - 结论是：前两段有效，第三段卡在更复杂密集场景。

3. `03_方法核查/`
   - 当第三课程卡住后，单独拆出技术核查线。
   - 目前已经确认：集中式 critic 改完整后，前期有帮助，但不足以单独解决后期退化。

4. `04_保留专门化/`
   - 这是接下来的新主线。
   - 当前理解是：问题不再只是 critic，而是单一 actor 从稀疏场景推到密集交互场景时容易破坏已有能力。
   - 所以下一步不再继续强推单一 actor，而是做“保留普通 actor + 引入密集 actor + 切换”的路线。

## 一句话总结

- `01` 是基础探索。
- `02` 是课程学习主线。
- `03` 是对卡点的技术核查。
- `04` 是基于前面结论转出的新方法主线。
