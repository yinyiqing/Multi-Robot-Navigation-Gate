# 多智能体共享 Policy 实验总结

> 历史记录：本文档前半部分记录早期两车共享 Policy 验证，后半部分补充三车共享 Policy Baseline。当前三车主线横向比较请以 `experiments/多智能体/三车主线实验矩阵.md` 和 `三车共享PolicyBaseline/test_multi_baseline_3_best_300episodes_summary.md` 为准。

## 结论

多智能体共享 policy 已经从“基本学不会”推进到“训练和测试都能跑通”。这说明多车环境、reset/goal 约束、测试流程和 ROS/Gazebo 端口隔离的调整是有效的。

这一版是当前的多智能体主线 baseline：同一个环境下多个机器人一起跑，并一起更新同一个 policy。

## 训练结果

- 训练版本：`multi-agent-shared-policy-v4`
- 训练日志：
  - `两车共享Policy验证/train_multi_detached_20260514_203649.log`
- 最佳评估结果：
  - `Epoch 9`
  - `success_rate = 0.800`
  - `collision_rate = 0.100`
  - `avg_reward = 86.616`
  - `avg_final_distance = 0.459`
- 训练过程中的趋势：
  - `Epoch 1`: `success_rate = 0.000`
  - `Epoch 2`: `success_rate = 0.200`
  - `Epoch 3`: `success_rate = 0.400`
  - `Epoch 4`: `success_rate = 0.450`
  - `Epoch 5`: `success_rate = 0.800`
  - `Epoch 7`: `success_rate = 0.650`
  - `Epoch 9`: `success_rate = 0.800`

训练中后期已经多次出现 `success=2/2`、`collision=0/2` 的 episode，说明不是单车偶然成功，而是两台车都能完成任务。

## 测试结果

- 早期测试日志：
  - `两车共享Policy验证/test_multi_detached_20260514_221005.log`
- 公平长测试日志：
  - `两车共享Policy验证/test_multi_baseline_fair300_detached_20260516_170905.log`
- 300 episode 公平长测试整体统计：
  - `success_rate = 0.792`
  - `collision_rate = 0.117`
  - `full_success_rate = 0.603`
  - `timeout_rate = 0.180`
  - `avg_reward = 86.951`
- 测试过程中 recent10 的代表性高点：
  - 一组达到：
    - `success_rate = 0.950`
    - `collision_rate = 0.050`
    - `full_success_rate = 0.900`
    - `avg_reward = 121.305`
  - 另一组达到：
    - `success_rate = 0.900`
    - `collision_rate = 0.050`
    - `full_success_rate = 0.800`
    - `avg_reward = 111.318`

判断：共享 policy 路线已经被验证可行，但 300 episode 公平测试也说明它不是最终最强版本。后续 dynamic reward weighted08 在同类测试中表现更好。

## 关键修改

- 保持共享 policy，不引入动态 reward。
- 保留 `random_near_obstacle = False`。
- 将多车环境改成更适合前期学习的弱耦合布局。
- 收紧 goal 采样范围和起始朝向，减少无意义的超难局。
- 缩短单个 episode 上限，减少卡死回合对数据的污染。
- 补充独立的多智能体 test 脚本，并修复 ROS/Gazebo 端口隔离问题。

## 当前结论

- 保留这版作为多智能体主线 baseline。
- 共享 policy 路线已经被验证是可行的。
- 后续动态 reward、更多机器人数量扩展和泛化测试，都应该在这版稳定 baseline 上继续做，而不是回到早期失败版本。

## 三车共享 Policy Baseline

在局部邻域 Critic 主线中，三车共享 Policy Baseline 作为公平对照重新训练并完成了 300 episodes 测试。

- 训练版本：`multi-agent-shared-policy-baseline-3-v1`
- 模型：`TD3_velodyne_multi_v4_shared_policy_3_best`
- 初始化：统一从 `TD3_velodyne_multi_v4` warm-start
- 训练日志：
  - `三车共享PolicyBaseline/train_multi_baseline_3_detached_20260520_171539.log`
- 300 episodes 测试日志：
  - `三车共享PolicyBaseline/test_multi_baseline_3_best_300episodes_clean.log`
- 测试摘要：
  - `三车共享PolicyBaseline/test_multi_baseline_3_best_300episodes_summary.md`

300 episodes 测试整体统计：

- `success_rate = 0.926`
- `collision_rate = 0.056`
- `full_success_rate = 0.797`
- `timeout_rate = 0.053`
- `avg_reward = 119.580`
- `avg_final_distance = 0.356`

这一结果说明三车共享 Policy Baseline 是强对照组。在当前三车弱耦合场景下，仅共享 policy 并从统一基础导航模型 warm-start，已经可以获得较高成功率。因此后续判断局部邻域 Critic 是否有效时，必须补齐 reward-only 与 weighted08 单变量对照，不能只用完整局部 Critic 方法直接对比共享 policy baseline。

## 下一步建议

- 保留当前多智能体版本作为第一版可复现实验结果。
- 将训练、测试日志和 checkpoint 关联起来归档。
- 后续可以继续尝试：
  - 动态 reward
  - 更多机器人数量扩展
  - 更严格的测试场景泛化验证
