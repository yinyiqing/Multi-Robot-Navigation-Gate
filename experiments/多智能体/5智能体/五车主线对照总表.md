# 五车主线对照总表

本文档记录五车标准场景下已经完成的主线对照实验，用于统一比较不同 reward 和 critic 设计在机器人数量增加后的表现。

## 统一口径

- 机器人数量：5
- 场景：`standard`
- warm-start：`TD3_velodyne_multi_v4`
- actor 执行输入：本车 24 维 observation
- 执行阶段通信：无
- 测试阶段 reward：individual reward，仅用于统计策略表现
- 测试规模：300 episodes
- 训练预算：A-H 为 20 epochs，测试使用各组 best checkpoint；Z0 不训练，直接测试原始 warm-start

## 实验定义

| 编号 | 方法 | 训练 reward | critic 训练输入 | 目的 |
| --- | --- | --- | --- | --- |
| Z0 | 原始 WarmStart ZeroShot | 无五车训练 | 测试阶段不使用 critic | 检查原始 warm-start 未经五车训练时的五车表现 |
| A | 共享 Policy Baseline | individual reward | 本车 observation + 本车 action | 五车基础对照 |
| B | RewardOnly | 可见邻居 cooperative reward | 本车 observation + 本车 action | 单独验证动态 reward |
| C | Weighted08 | `0.8 * own + 0.2 * distance-weighted neighbor` | 本车 observation + 本车 action | 验证距离加权 reward shaping |
| D2 | 几何邻域 Critic + Weighted08 | Weighted08 | 本车 observation + 本车 action + 邻居几何 + mask | 验证几何 critic 与 Weighted08 的组合 |
| E | 纯几何邻域 Critic | individual reward | 本车 observation + 本车 action + 邻居几何 + mask | 单独验证几何 critic |
| F | Weighted09 | `0.9 * own + 0.1 * distance-weighted neighbor` | 本车 observation + 本车 action | 降低邻居 reward 权重后的对照 |
| G | 几何邻域 Critic + RewardOnly | 可见邻居 cooperative reward | 本车 observation + 本车 action + 邻居几何 + mask | 验证 RewardOnly 与几何 critic 的组合 |
| H | Weighted09 Active | `0.9 * own + 0.1 * distance-weighted active-neighbor` | 本车 observation + 本车 action | 验证 inactive/done 机器人是否污染 cooperative reward |

## 关键观察

- Z0 明显低于 A/B/E/F，说明五车训练确实改善了原始 warm-start 的五车适应性。
- Z0 的 avg_env_steps 明显偏高，300-step episode 比例达到 0.303，说明原始模型在五车场景中更容易出现犹豫、绕行慢或局部死锁。
- Baseline 与 H 是五车标准场景下 full_success_rate 最高的方法。
- RewardOnly 与 baseline 接近，个体成功率略高、碰撞率更低，但全成功率略低。
- Weighted08 明显降低碰撞率，但 success_rate 和 full_success_rate 下降，avg_env_steps 大幅增加，说明策略更保守、完成更慢。
- D2 没有超过 Weighted08 和 baseline，说明几何 critic 没有抵消 Weighted08 带来的保守性。
- E 接近 baseline，且明显好于 C/D2 的 full_success_rate，说明几何 critic 单独不是主要问题。
- F 明显优于 Weighted08，说明降低邻居 reward 权重可以缓解保守性；但 F 仍略低于 baseline，优势不充分。
- G 没有超过 B RewardOnly，且 collision_rate 明显升高，说明几何邻域 critic 与 RewardOnly 的组合不稳定。
- H 与 baseline 的 full_success_rate 持平，collision_rate 更低，但 avg_env_steps 更高，且 300 episodes 中 timeout_episode_rate 为 0.233，说明它更安全但长尾完成效率不足。
- H 训练日志中 active-neighbor cooperative reward 几乎没有触发，说明旧 F 的 cooperative reward 很可能被 inactive/done 机器人污染；去掉这类污染能降低碰撞，但还没有带来稳定协同优势。

## 当前判断

五车标准场景下，训练后的 A/B/E/F/H 明显优于 Z0，说明五车训练本身是有效的。主要问题不是“有没有从 warm-start 学到东西”，而是不同多机 reward 和 critic 设计是否能在五车规模下稳定超过共享 Policy baseline。

Weighted08 的邻居 reward 权重在五车规模下偏强，削弱了个体目标驱动力。Weighted09 已证明降低邻居 reward 权重有效；H 进一步说明 inactive/done 邻居会污染 cooperative reward，但过滤后协同信号几乎消失，仍不能稳定超过 baseline。RewardOnly 单独有效，但与几何邻域 critic 组合后没有收益。后续如果继续优化五车，优先方向应是训练稳定性、neighbor exposure 设计和更可靠的 reward/critic 消融，而不是简单加大邻居 reward。
