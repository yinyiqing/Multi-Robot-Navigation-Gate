# 局部邻域 Critic 实验总结

> 历史记录：本文档主要总结原始 D：三车局部邻域 Critic + Weighted08。当前主方法候选 D2：几何邻域 Critic + Weighted08 已另行归档，最新横向比较请以本目录 `README.md` 与 `experiments/多智能体/三车主线对照总表.md` 为准。

## 实验目的

本实验验证一种基于 CTDE 思想的多机器人导航训练方式：执行阶段 actor 只使用本车观测，训练阶段 critic 额外接收雷达范围内的局部邻居信息。目标是在不引入执行阶段通信的前提下，提高共享 policy 在多机器人交互场景中的价值估计质量和协同稳定性。

## 方法设置

actor 输入保持原 TD3 导航结构：

```text
actor_input_i = obs_i
state_dim = 24
action_dim = 2
```

critic 在训练阶段接收本车状态、动作和局部邻域 context：

```text
critic_input_i = [obs_i, action_i, neighbor_context_i]
```

邻居 context 按距离从近到远排序，最多预留 9 个邻居槽位。每个邻居包含：

```text
relative_x
relative_y
distance
bearing
neighbor_linear_action
neighbor_angular_action
visible_mask
```

因此 critic 的状态/context 分支为：

```text
24 + 9 * 7 = 87
```

不存在或超出雷达范围的邻居用 0 补齐。该设计使 2 车、3 车、5 车和未来 10 车可以共用同一套 critic 输入结构。

## 三车训练

训练脚本：

```text
scripts/start_training_detached_multi_local_critic_3.sh
```

训练日志：

```text
三车多邻居验证/train_multi_local_critic_3_detached_20260520_140216.log
```

模型前缀：

```text
TD3_velodyne_multi_v4_local_critic_3
```

本轮训练使用共享 policy baseline 的 actor warm-start，critic 因输入维度变化重新初始化。训练 10 个 epoch 后停止，best checkpoint 出现在 epoch 8：

```text
Epoch 8
success_rate = 0.933
collision_rate = 0.017
avg_reward = 122.253
avg_final_distance = 0.250
```

训练评估曲线显示，三车局部邻域 Critic 在前 6 到 8 个 epoch 内快速收敛。epoch 9 和 epoch 10 相比 epoch 8 略有回落，因此后续测试使用 best checkpoint，而不是 latest checkpoint。

## 三车 best checkpoint 测试

测试模型：

```text
TD3_velodyne_multi_v4_local_critic_3_best
```

测试规模：

```text
N = 3
episodes = 300
```

测试阶段使用 individual reward 统计策略表现，不使用训练阶段的 reward shaping 作为评价指标。

整理后的有效测试记录：

```text
三车多邻居验证/test_multi_local_critic_3_best_300episodes_clean.log
```

测试摘要：

```text
三车多邻居验证/test_multi_local_critic_3_best_300episodes_summary.md
```

最终结果：

| 指标 | 数值 |
| --- | ---: |
| `success_rate` | `0.913` |
| `collision_rate` | `0.052` |
| `full_success_rate` | `0.747` |
| `timeout_rate` | `0.100` |
| `success_80_rate` | `0.747` |
| `avg_reward` | `117.709` |
| `avg_final_distance` | `0.352` |
| `avg_episode_env_steps` | `72.3` |

累计计数：

```text
single-agent success = 822 / 900
single-agent collision = 47 / 900
full-success episode = 224 / 300
success_80 episode = 224 / 300
timeout episode = 30 / 300
```

其中 `success_rate` 按单个机器人任务统计，`full_success_rate` 按 episode 统计：只有三台机器人都成功到达目标时，该 episode 才记为 full success。

## 工程修正

三车测试过程中发现目标点采样在少数 reset 中可能无法满足固定 clearance 约束。该问题属于环境采样鲁棒性问题，不是策略失败。后续已在环境中增加 reset 级别重试和目标 clearance 兜底：

```text
1.2 -> 1.0 -> 0.8
```

同时清理了 detached 测试的 ROS/Gazebo 残留进程，避免同一端口上存在多套 Gazebo 实例导致反复 respawn。

## 阶段判断

本轮三车实验说明局部邻域 Critic 机制已经跑通，并且在三车场景下可以取得较高的单机器人成功率和可用的 full success rate。该结果支持继续推进公平对照实验，但尚不能单独作为论文结论。

下一步需要在三车场景下补齐同预算对照：

1. 共享 policy baseline
2. reward-only / weighted08
3. 局部邻域 Critic

三组实验需要使用相同初始化、训练预算、测试 episode 数和评价指标。只有在公平对照下，才能判断提升主要来自 critic 的局部邻居 context，而不是 warm-start、训练随机性或 reward shaping。
