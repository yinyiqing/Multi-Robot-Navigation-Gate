### 1. 为什么改这条线

- 旧问题：五车复杂阶段里，actor 一解冻就容易退化
- 新判断：不能只怪 actor，本质上也可能是 critic 没真正建模多机耦合
- 因此先不继续乱调解冻，而是先把 critic 改得更接近 MADDPG 式 joint-action critic

### 2. 这次改了什么

- `ReplayBuffer` 新增多机 transition 信息：
  - `joint_states`
  - `joint_actions`
  - `joint_next_states`
  - `active_mask`
  - `next_active_mask`
  - `agent_index`
- `train_local_critic()` 支持可选 `joint-action critic`
- 更新当前 agent actor 时：
  - 当前 agent 动作用当前 actor 重算
  - 其他 agent 动作用 replay 里的旧动作
  - 等价于把其他 agent 动作当固定背景

## 2026-07-05

### 14. 第二轮代码核查：mask / target / actor更新路径

- 这一步不是新跑实验
- 目标是确认：
  - joint-action critic 代码是不是按我们以为的方式在训练
  - 有没有明显实现错误会把结论带偏

### 15. 当前确认没问题的点

- replay buffer 里已经存了：
  - `joint_actions`
  - `joint_next_states`
  - `active_mask`
  - `next_active_mask`
  - `agent_index`
- actor 更新时：
  - 当前 agent 的动作由当前 actor 重算
  - 其他 agent 的动作直接来自 replay buffer
  - 没有把其他 agent 的 actor 梯度一起带进来
- 所以老师提到的“其他 actor 要不要 detach”这个点
  - 在当前这版 joint-action critic 里，处理方向是对的

### 16. 当前发现并已修正的一个关键细节

- target joint action 的构造还不够标准：
  - 现在是先对所有 agent 用 `actor_target` 生成 joint next action
  - 原来 TD3 的 target smoothing noise 只明确加在“当前样本 agent”的 `next_action` 上
  - 其他 agent 的 next action 还是无噪声的 target 动作
- 现在已经修正为：
  - 所有 agent 的 target action 都统一经过 TD3 风格的 smoothing noise
  - inactive agent 再按 `next_active_mask` 置零
- 这样 target critic 看到的 joint next action 更一致，也更接近标准 joint-action target

### 17. 当前阶段结论

- 目前没发现“actor 更新路径写错了，所以之前全白跑了”这种大问题
- 当前更像是：
  - joint-action critic 主体方向是对的
  - target 细节里确实有一处不够标准，但现在已经补上了
- 因此下一步最合理的是：
  - 直接做一组稍长验证
  - 看修正后的 joint-action critic 能不能继续稳住，而不是立刻跳去 BC
