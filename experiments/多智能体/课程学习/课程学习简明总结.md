# 课程学习简明总结

## 目标

课程学习这条线的目标是分阶段提升多机器人导航能力：

1. 先补稳单车局部导航。
2. 再把单车能力接回多车普通场景主线。
3. 最后尝试解决人工设计的密集多车交互。

## 已完成阶段

### 第一课程：单车局部导航

这一阶段已经完成。

主要解决的问题：

- 墙边振荡
- 目标贴墙难以收敛
- 近目标停滞
- 局部碰撞尾部失败

当前保留的两个模型：

- `stage1g best`：综合最稳，作为后续 warm start 主入口
- `stage1i best`：hard case 更强，作为补充对照

结论：第一课程达到了预期，后续不再继续扩展单车课程。

### 第二课程：多车主线回接

这一阶段的目标不是密集交互，而是把第一课程得到的 actor 接回多车普通场景主线。

主线顺序：

`stage1 -> 2A -> 2D -> 3A -> 3D -> 3D2 -> 5A -> 5D`

当前结果：

- 三车普通场景中，`课程 3D2 best` 是当前最好节点
  - agent success `0.939`
  - collision `0.046`
  - full success `0.830`
- 五车普通场景中，`课程 5A best` 是当前最稳节点
  - success `0.897`
  - collision `0.087`
  - full success `0.600`
- `5D` 相比 `5A` 没有继续提升

结论：第二课程已经完成“主线回接”，课程 actor 可以迁移到普通 2 车、3 车、5 车场景。

## 未完成阶段

### 第三课程：人工密集多车交互

这一阶段还没有成功。

已经做过：

- 三车轻密集诊断
- 三车过渡课程尝试
- 三车人工密集直接训练
- 五车人工密集诊断
- 五车 `dense_gentle`
- 五车 `dense_bridge`

当前观察：

- 普通多车场景中有效的模型，不能自然迁移到人工密集交互场景
- 五车共享 policy 在普通场景可用，但在强对称、强冲突场景下容易一起冲向冲突区
- 即使加入交互 reward，五车密集课程仍未学成稳定让行规则
- 当前准备按“非对称密集课程”重做第三课程，先教模型在有差别的冲突中学会先后顺序，再逐步逼近强对称密集场景

五车人工密集诊断结果：

- success `0.400`
- collision `0.617`
- full success `0.075`

结论：人工密集强交互仍是当前主要未解决问题。

## 当前总判断

目前课程学习的进展可以概括为：

- 第一课程成功：单车局部导航已补稳
- 第二课程成功：普通多车主线已接回
- 第三课程未完成：人工密集多车交互仍未解决

## 关键现象

目前多次出现同一个现象：

- 训练早期 `best` 较好
- actor 解冻后继续训练，性能反而退化

这说明当前问题不只是 warm start 是否有效，更可能与以下因素有关：

- 新阶段数据分布变化较大
- critic 尚未稳定时就开始推动 actor 更新
- 密集交互场景本身对共享 policy 的协调要求过高

## 当前可用结论

如果只看当前已经可靠得到的结果，可以给出两点：

1. 课程学习对单车到普通多车的迁移是有效的。
2. 课程学习尚未解决人工密集多车交互问题。
