# 第三课程：多车密集交互

## 这门课解决什么

第三课程才是人工设计多车交错、靠近、会车、目标聚集等密集交互场景。

它和第二课程不同：

- 第二课程：先把主线 A/D/D2 接回来，确认课程 actor 能进入 2车和 3车普通场景。
- 第三课程：在主线节点稳定后，再用手工密集 case 专门补多车交互短板。

## 后续入口

第三课程已经用课程 3D2 best 做完第一轮轻密集诊断。结果说明：普通三车 standard 上的提升，没有自然迁移到密集交互。

当前建议：

1. 暂停第三课程训练，不再继续加 case。
2. 先查课程 3D2 后段为什么从 epoch 10 开始退化。
3. 查 local critic 邻居 context 在训练过程中是否真的有效。
4. 当前主线保留课程 3D2 best，不使用 latest。
