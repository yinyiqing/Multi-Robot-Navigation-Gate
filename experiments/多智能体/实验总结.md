# 多智能体实验总结

## 结论

多智能体共享 policy 已经从“基本学不会”提升到“训练和测试都能稳定成功”，说明本次多车环境简化、reset/goal 约束调整和 test 流程修复是有效的。

## 训练结果

- 训练版本：`multi-agent-shared-policy-v4`
- 训练日志：
  - `train_multi_detached_20260514_203649.log`
- 最佳评估结果：
  - `Epoch 9`
  - `success_rate = 0.800`
  - `collision_rate = 0.100`
  - `avg_reward = 86.616`
  - `avg_final_distance = 0.459`
- 训练过程中的趋势：
  - `Epoch 1`: `success_rate = 0.000`
  - `Epoch 2`: `success_rate = 0.200`
  - `Epoch 3`: `success_rate = 0.400`
  - `Epoch 4`: `success_rate = 0.450`
  - `Epoch 5`: `success_rate = 0.800`
  - `Epoch 7`: `success_rate = 0.650`
  - `Epoch 9`: `success_rate = 0.800`
- 训练中后期已多次出现 `success=2/2`、`collision=0/2` 的 episode，说明不是单车偶然成功，而是两车都能完成任务。

## 测试结果

- 测试日志：
  - `test_multi_detached_20260514_221005.log`
- 最近 10 个 episode 的代表性统计：
  - 一组达到：
    - `success_rate = 0.950`
    - `collision_rate = 0.050`
    - `full_success_rate = 0.900`
    - `avg_reward = 121.305`
  - 另一组达到：
    - `success_rate = 0.900`
    - `collision_rate = 0.050`
    - `full_success_rate = 0.800`
    - `avg_reward = 111.318`
- 说明：
  - `success_rate` 已经很高，代表按机器人统计时大部分都能成功到点
  - `full_success_rate` 达到 `0.8~0.9`，代表整局两台车同时成功的比例也很高
  - `collision_rate` 很低，说明策略不是靠冒险硬冲得到的

## 关键修改

- 保持共享 policy，不引入动态 reward
- 保留 `random_near_obstacle = False`
- 多车环境改为更适合前期学习的弱耦合布局
- 收紧 goal 采样范围和起始朝向，减少无意义的超难局
- 缩短单个 episode 上限，减少卡死回合对数据的污染
- 补充独立的多智能体 test 脚本，并修复 ROS/Gazebo 端口隔离问题

## 当前结论

- 这版多智能体已经可以作为当前主线 baseline
- 共享 policy 路线被验证是可行的
- 后续若继续做研究尝试，建议在这版稳定 baseline 上开展，而不是回到早期失败版本

## 下一步建议

- 保留当前多智能体版本作为第一版可复现实验结果
- 将本次训练与测试曲线、日志、checkpoint 一起归档
- 下一步可以尝试：
  - 动态 reward
  - 更多机器人数量扩展
  - 更严格的测试场景泛化验证
