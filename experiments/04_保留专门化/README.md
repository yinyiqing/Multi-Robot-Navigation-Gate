# 保留专门化

这里放从“保留旧能力”一路收敛到“局部交互增强”的主线。

## 这条线回答什么

一句话：

**如何把已经学会普通导航的多机器人策略扩展到密集交互场景，同时不破坏它原来已经学好的能力。**

## 为什么会转到这里

前面的结论已经比较清楚：

1. 课程学习前两段有效：
   - 单车补稳有效
   - 普通多车回接有效
2. 到更复杂的密集多车阶段后：
   - warm start 模型本身还能跑
   - 但 actor 一继续更新，性能容易退化
3. 集中式 critic 核查后发现：
   - critic 改完整后前期更好
   - 但长期退化还在

所以现在的判断是：

**主问题不再只是 critic，而是单一 actor 在普通导航和密集协同之间可能存在能力冲突。**

## 现在的主线收敛

目前这条线不再把“两个 actor 切换”当最终目标。

现在更顺的主线是：

1. 先证明单一 actor 继续覆盖训练不划算
2. 再证明不同阶段策略确实存在能力偏向
3. 再验证“粗双 actor 切换”并不够好
4. 最后把方法收敛到：
   - 保留原有主干能力
   - 只在强交互局部状态下，做轻量交互增强

也就是：

**从 preserve-and-specialize 的直觉出发，最后收敛到局部交互感知的门控注意力增强。**

## 这一目录后面准备怎么放

- `01_冲突验证/`
  - 先证明单一 actor 继续往 dense 训练时，会不会破坏原有普通场景能力
- `02_双actor切换/`
  - 作为中间验证：能力偏向存在，但粗切换不够
- `03_门控注意力增强/`
  - 主线收敛方向：局部交互感知 + 轻量注意力增强
- `04_安全兜底/`
  - 如果主线有效，再把传统规划或安全控制并进来

现在目录已经先建好，后面所有新实验都优先往这 4 个子目录里放。

## 当前一句话记录

- 旧主线卡点：actor 解冻后退化
- critic 核查结论：有帮助，但不够
- 新主线：保留旧 actor，不再覆盖式训练单一 actor

## 故事思路 vs 验证情况

### 1. 单一 actor 继续往 dense 训练，不一定是纯增益

- 故事思路：
  - 如果继续覆盖式训练同一个 actor，它可能会破坏原来已经学好的普通导航能力
- 当前验证：
  - 基本支持
  - `THREE_MID -> 标准五车` 明显差于 `5A`
  - `THREE_MID -> dense` 也没有变得更强
- 当前结论：
  - overwrite 路线先停

### 2. 不同阶段训出来的 actor，能力重心不同

- 故事思路：
  - 普通场景训出来的 actor 更偏普通导航
  - 密集场景训出来的 actor 更偏密集交互
- 当前验证：
  - 基本支持
  - `5A` 在标准五车最稳
  - 旧 `PAIR(from_5a)` 在 dense 上明显强于 `5A`
  - 新 `PAIR(from_5d)` 在训练端已经优于旧 `PAIR(from_5a)`
  - `5D` 介于两者之间，更像过渡 actor
- 当前结论：
  - 现在更像“能力偏向不同”，而 `5D -> PAIR` 是更合理的专门化链路

### 3. preserve-and-specialize 比 overwrite 更合理

- 故事思路：
  - 不强行把一个 actor 训成全能
  - 而是保留普通 actor，再准备一个更适合 dense 的 actor
- 当前验证：
  - 主干已经成立
  - `5A` 适合作为普通 actor
  - `PAIR` 适合作为当前密集 actor
  - `THREE_MID` 证明 overwrite 不划算
- 当前结论：
  - 可以进入更细粒度的“按交互状态增强”实验

### 4. 现在这条故事怎么讲最稳

- 不建议讲成：
  - 普通能力和 dense 能力完全冲突，必须彻底拆成两个策略
- 更建议讲成：
  - 单一 actor 的继续覆盖训练不稳定，而且会伤到原有能力
  - 不同阶段得到的 actor 呈现出不同能力偏向
  - 粗双 actor 切换说明“策略级硬切换”太粗
  - 所以最后收敛到“局部交互感知的门控增强”更合理

### 5. 当前先定的候选组合

- 普通 actor：
  - `5A`
- 密集 actor：
  - `PAIR(from_5d)`
- 备用强基线 / 过渡 actor：
  - `5D`
