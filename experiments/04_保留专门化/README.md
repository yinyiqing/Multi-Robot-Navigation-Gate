# 保留专门化

这里放新的双 actor 主线。

## 这条线回答什么

一句话：

**如何把已经学会普通导航的多机器人策略扩展到密集交互场景，同时不破坏它原来已经学好的能力。**

## 为什么会转到这里

前面的结论已经比较清楚：

1. 课程学习前两段有效：
   - 单车补稳有效
   - 普通多车回接有效
2. 到更复杂的密集多车阶段后：
   - warm start 模型本身还能跑
   - 但 actor 一继续更新，性能容易退化
3. 集中式 critic 核查后发现：
   - critic 改完整后前期更好
   - 但长期退化还在

所以现在的判断是：

**主问题不再只是 critic，而是单一 actor 在普通导航和密集协同之间可能存在能力冲突。**

## 现在的最小主线

- 普通 actor：保留已经学好的普通导航能力
- 密集 actor：专门处理高交互密集场景
- 切换规则：根据局部交互强度在两者之间切换

第一版先不强行把传统规划一起并进来。
先验证双 actor 的 preserve-and-specialize 主线能不能站住。

## 这一目录后面准备怎么放

- `01_冲突验证/`
  - 先证明单一 actor 继续往 dense 训练时，会不会破坏原有普通场景能力
- `02_双actor切换/`
  - 普通 actor + 密集 actor 的最小切换版
- `03_安全兜底/`
  - 如果双 actor 主线成立，再把传统规划或安全控制并进来

现在目录已经先建好，后面所有新实验都优先往这 3 个子目录里放。
