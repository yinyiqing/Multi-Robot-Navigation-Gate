# 03 门控注意力增强

这一步开始把主线从“粗双 actor 切换”收敛到“局部交互感知的轻量增强”。

目前按导师反馈，优先级已经调整：

- 先做门控
- 注意力作为后续可选增强

## 这一步回答什么

一句话：

**不是直接切整个策略，而是在强交互局部状态下，只给原有策略补一层轻量交互增强。**

## 为什么转到这里

前面已经看到：

1. 单一 actor 继续覆盖训练不划算
2. 不同阶段策略确实有能力偏向
3. 但粗双 actor 切换两边都没赢

所以现在的判断是：

**问题不只是“要不要多策略”，而是“能不能识别当前交互状态，并只在需要时增强交互建模”。**

## 当前最小主线

- 冻结普通 actor：`5A`
- 冻结 dense actor：`PAIR(from_5d)`
- 单独训练一个轻量门控网络
- 让门控根据局部交互状态决定更偏向哪个 actor
- 如果门控有效，再考虑是否加注意力特征

## 当前建议顺序

1. 先补齐 `PAIR(from_5d)` 的正式 test
2. 再做“冻结双 actor + 单独训练门控网络”
3. 先验证门控是否能同时保住普通能力和 dense 能力
4. 门控有效后，再试注意力是否还能继续提升
